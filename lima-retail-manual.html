01 — Fundamentos
Tokens de color

02 — Fundamentos
Tipografía
Sistema tipográfico sin dependencias externas. Pesos limitados a 400, 500 y 600 — nunca 700 o superior para mantener sobriedad. Tamaño base 13px.

12px / 400 / text-2

11px / 500 / muted

--font: -apple-system, BlinkMacSystemFont, "Segoe UI", "Inter", Helvetica, sans-serif;

/* Pesos permitidos: 400 · 500 · 600 — nunca 700+ */
.panel-title  { font-size: 14px; font-weight: 600; letter-spacing: -.01em; }
.kpi-val       { font-size: 24px; font-weight: 600; letter-spacing: -.02em; }
.kpi-lbl       { font-size: 10px; font-weight: 600; text-transform: uppercase; letter-spacing: .07em; }
.sec-lbl       { font-size: 10px; font-weight: 600; letter-spacing: .09em; text-transform: uppercase; }
03 — Fundamentos
Layout Shell
Estructura de dos columnas: sidebar oscuro fijo (240px) + área principal. El topbar blanco es sticky dentro del main y separa la navegación del contenido.
sidebar
240px
topbar · background: #fff · border-bottom
.content · padding: 22px 28px · flex-direction: column · gap: 22px
--r-lg
14px — modales
--r
12px — cards, panels
--r-sm
8px — botones, inputs
--r-xs
6px — chips, badges
HTML 
<div class="shell">
  <aside class="sidebar">
    <div class="s-brand">
      <div class="s-title">Nombre de la herramienta</div>
      <div class="s-sub">Agencia Lima Retail</div>
    </div>
    <!-- .s-item para navegación -->
  </aside>

  <main class="main">
    <div class="topbar">
      <div class="topbar-title">Título de sección</div>
    </div>
    <div class="content">
      <!-- contenido -->
    </div>
  </main>
</div>
04 — Componentes
Botones
Cuatro variantes. El botón primario usa --brand. Nunca usar font-weight: 700. Los iconos SVG son opcionales y van al inicio.
HTML 
<button class="btn">Default</button>
<button class="btn primary">Primary</button>
<button class="btn danger">Danger</button>
<button class="btn ghost">Ghost</button>
<button class="btn" disabled>Disabled</button>
05 — Componentes
Formularios
Dos contextos: inputs sobre fondo claro (app) e inputs sobre fondo oscuro (sidebar). Los presets son botones pill que actualizan el valor del input.
Input claro (app)
S/
Input oscuro (sidebar)
Costo por conversación
HTML 
<!-- Input claro -->
<input class="inp" type="number" placeholder="0.00" />

<!-- Con prefijo S/ -->
<div class="inp-wrap">
  <span class="inp-pre">S/</span>
  <input class="inp" type="number" />
</div>

<!-- Input oscuro (sidebar) con presets -->
<label class="sf-lbl">CPL</label>
<input class="sf-input" type="number" />
<div class="sf-presets">
  <button class="sf-preset on">S/3</button>
  <button class="sf-preset">S/5</button>
</div>

09 — Componentes
Insight Boxes
Alertas con borde lateral de color. Cuatro variantes: informativa, positiva, advertencia y error. Apilables, separadas por margin-top: 8px.
Información: El archivo fue procesado correctamente con 266 líneas de venta.
Éxito: La proyección muestra un ROI positivo de 403.7% sobre la inversión.
Atención: El CPL seleccionado es alto — considera optimizar la segmentación.
Error: No se pudo cargar el archivo. Verifica el formato e inténtalo de nuevo.
HTML 
<div class="insight info"><b>Información:</b> Texto…</div>
<div class="insight ok"><b>Éxito:</b> Texto…</div>
<div class="insight warn"><b>Atención:</b> Texto…</div>
<div class="insight err"><b>Error:</b> Texto…</div>
